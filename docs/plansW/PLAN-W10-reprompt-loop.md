# PLAN-W10 — Reprompt Loop (The "Cursor for Video Editing" Feature)

> **Problem**: The core value proposition — "tell the AI to zoom in at 0:50, and it does" — doesn't exist yet. There's no way for a user to review an edit plan and ask for changes.
> **Goal**: Chat sidebar where users can reprompt the AI agent, which re-plans and updates the timeline. This is the feature that makes FlowStudio "Cursor for video editing."

---

## User Experience

### Step 1: AI generates initial edit plan
User records → pipeline runs → edit plan appears on timeline.

### Step 2: User reviews and gives feedback
In the chat sidebar (InspectorPanel), the user types:
```
"Zoom in more deeply around 0:50 when I click the button"
"Remove the dead time between 0:30 and 0:45"  
"Speed up the typing section from 1:00 to 1:20"
"Add a transition at 0:55"
```

### Step 3: AI re-plans
The reprompt goes to the Railtracks gateway:
```
POST /api/v1/reprompt
{
  "project_id": "...",
  "previous_edit_plan": [...current edits...],
  "feedback": "Zoom in deeply around 0:50 on the button click"
}
```

### Step 4: New edit plan version applied
Response contains updated edit plan (v2). Timeline smoothly transitions to show new edits.

### Step 5: Iterate
User can keep reprompting. Each creates a new version. User can undo to previous version.

```typescript
interface ChatMessage {

async function sendReprompt(message: string) {
  const currentPlan = editorContext.getAiEditPlan();

  const response = await fetch(`${RAILTRACKS_GATEWAY_URL}/api/v1/reprompt`, {
    method: "POST",
    body: JSON.stringify({
      project_id: projectId,
      previous_edit_plan: currentPlan,
      feedback: message,
    }),
  });

  const result = await response.json();
  editorContext.applyEditPlan(result.edit_plan, result.version);

  return {
    role: "assistant",
    content: formatEditPlanSummary(result.edit_plan, currentPlan),
    editPlanVersion: result.version,
  };
}
```

### Edit Plan Summary (what the AI "says back")

Instead of just "done", the assistant message describes what changed:
```
"I've made 2 changes to the edit plan:
 • Added a 2x zoom at 0:48–0:52 targeting the button area
 • Adjusted the cut at 0:55 to preserve the click animation

Version 2 applied to your timeline."
```

This summary is generated by diffing the old and new edit plans.

## The Key Question: Does the AI Actually Understand?

When the user says "zoom in at 0:50 on the button", the AI needs:
1. The full signal context (what happened at 0:50 in the recording)
2. The current edit plan
3. The user's feedback

The Railtracks reprompt endpoint already handles this:
```python
@rt.function_node
async def reprompt_pipeline(previous_plan: list, feedback: str):
    revised = await rt.call(EditAgent, format_reprompt(previous_plan, feedback))
    return revised
```

The `format_reprompt` function constructs a prompt that includes:
- Current edit plan (with time ranges)
- User feedback
- System prompt instructing the agent to modify specific edits

**This is where the magic happens**: The AI reads the edit plan, understands the time codes, and makes targeted modifications.

## Example Interaction

**User**: "The zoom at 0:50 should be slower and deeper"

**AI receives**:
```
Current edit plan:
- zoom: 49000ms–52000ms, zoomLevel: 1.5, easing: ease-in-out

User feedback: "The zoom at 0:50 should be slower and deeper"
```

**AI responds with updated plan**:
```json
[
  { "editType": "zoom", "sourceStartMs": 48000, "sourceEndMs": 54000,
    "params": { "zoomLevel": 2.5, "easing": "ease-in-out", "duration": 6000 } }
]
```
